# CV20.DS6.US4 — Traverse Change Request Cycles

**Type:** User Story  
**Status:** 🟡 Planned

## Story

As Navigator, I want Builder to work through one Change Request at a time with
confirmation, short planning, implementation, validation evidence, and done note
so that refinement remains fast without becoming invisible improvisation.

## Outcome

A selected CR can move through confirm, plan, implement, validate, and done note
with visible surfaces and appropriate pauses for Navigator judgment.

## Notes

The CR cycle should be lighter than full Delivery Work, but still explicit
where scope, validation, or user-visible behavior is involved.
