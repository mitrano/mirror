# v0.27.1 — Windows UTF-8 Repair Command

## Digest

Mirror Mind v0.27.1 adds an explicit, backup-gated repair command for reversible
Portuguese mojibake in local SQLite user text, preserving the Windows UTF-8
compatibility work in a reusable update-safe path.

## Highlights

- Added `uv run python -m memory repair-encoding` as a dry-run scanner.
- Added `--apply` with automatic database backup before mutation.
- Repairs reversible Latin-1/Windows-1252 mojibake such as `Ã©` → `é` and

- Documented the repair in troubleshooting and runtime repair policy.

## Narrative Context

A local Windows installation surfaced journey descriptions with mojibake after
runtime updates. The CLI already prefers UTF-8 stdio, so the remaining issue was
historical data that had been stored incorrectly in the database. A one-off fix
solved the local symptom, but future updates needed a repeatable, documented,
safe repair lane.

## What Changed

- New command: `python -m memory repair-encoding [--mirror-home PATH] [--apply]`.
- The command scans known user-text columns in the Mirror database and previews
  each reversible repair.
- The apply path creates a backup first and aborts if backup fails.
- Tests cover Latin-1 style mojibake, Windows-1252 uppercase mojibake, safe
  handling of `Âncora`, database scan/apply, and top-level CLI dispatch.

## Conscious Exclusions

- No automatic repair during startup or `runtime update`; user text mutation must
  remain explicit.
- No attempt to guess irrecoverable replacement-character corruption.
- No broad mutation of arbitrary extension tables.

## Lessons

Encoding compatibility has two layers: future stdio/file correctness and repair
of historical stored text. The first belongs in runtime defaults; the second
belongs in an explicit, reviewable, backup-gated command.

## Next Horizon

If future Windows compatibility issues appear, add them to the same repair policy
instead of creating ad hoc one-off scripts.
