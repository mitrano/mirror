# v0.10.10 — Pending Release Notes Prompt

**Date:** 2026-05-25

## Highlights

- Changed the welcome update prompt from ambiguous `what changed?` wording to
  `show pending release notes`.
- Updated the release-notes skill so questions about a newly available version
  prefer `runtime release-notes pending`.
- Preserved explicit `latest` and `vX.Y.Z` release-note flows for users who ask
  for a single release.

## Why This Patch Exists

Production validation of v0.10.8 showed release detection working: opening Mirror
surfaced v0.10.9 automatically. But the suggested question, `what changed?`, was
too ambiguous. The agent interpreted it as a working-tree question and inspected
git status instead of showing release notes.

## What Changed

The welcome now suggests:

```text
Ask: "show pending release notes" or "update my Mirror"
```

The Pi release-notes skill now maps natural-language questions about the newly
available Mirror version to:

```bash
uv run python -m memory runtime release-notes pending
```

That is the correct final-user answer because it includes every release between
the installed version and the current stable version.

## Next Horizon

After production updates to this version, publish a later stub release to validate
the full loop: welcome detects a new release, suggests pending release notes, and
the user's question renders the cumulative notes instead of git status.
