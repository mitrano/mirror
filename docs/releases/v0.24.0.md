# v0.24.0 — Soul Mode First Ritual

**Date:** 2026-06-08

## Highlights

- Adds `☾ Soul Mode` activation through `/mm-soul [slug]` and `python -m memory soul load [slug]`.
- Renders the Soul Mode entry surface with the inward-day invitation.
- Adds Possible Listenings with situated Self, Shadow, Wisdom, and Beauty options.
- Implements first-release listening lenses for Self Voice and Shadow Voice.
- Keeps the user conversing with Mirror while voice responses appear inside ritual cards.
- Adds Fruit In Maturation state so one provisional harvest can be refined across turns.
- Adds Harvested Fruit and explicit save-or-decline journal confirmation.
- Saves Soul Mode harvests as structured Markdown journal entries with origin metadata.
- Injects the user's `self/soul` identity layer into the composed Self Voice prompt.
- Adds a Soul-to-Builder boundary so Soul Mode does not mutate project state directly.
- Improves web visibility for Journal memories, formatted journal detail, and conversation maintenance.

## Where We Started

Soul Mode began as an exploration of ritual and visual grammar for helping the
user remember who they are. The early question was not how to add another chat
command, but how to create a passage of state: from ordinary reporting, into
inner listening, through a voice or rite, toward a fruit that could be kept.

The product risk was overbuilding. Soul Mode has a broad horizon: Wisdom Voice,
Beauty Voice, Passagem, Return To Center, literary and philosophical fragments,
richer curation, psyche-layer integration, and possible web surfaces. The first
release needed to prove the core grammar without pretending the whole ecosystem
was complete.

## What Changed

Soul Mode is now an explicit operating lens. Activating it renders a ritual entry
surface with the line `Soul Mode turns the day toward the inner life`, the horizon
`✦ IN ORDER TO / remember who you are`, and the opening question `how is your day
going today?`. The mode is represented in the lifecycle and status line as `☾
Soul Mode`.

The opening conversation now has a visible threshold. Thin day reports are held
as living field without forcing a menu. When the user moves from events into
inner contract, fear, wound, desire, compulsion, discomfort, or threatened
belonging, Mirror can render `✧ POSSIBLE LISTENINGS` with situated options. The
card ends by telling the user they can hear one of the voices now or continue the
conversation.

The first implemented voices are Self and Shadow, but they are not separate chat
characters. They are listening lenses. When the user chooses one, Mirror renders
a card such as `✦ SELF VOICE LISTENING` or `◐ SHADOW VOICE LISTENING`; the voice
response appears inside the card, and Mirror then bridges that response back into
the conversation. This avoids splitting the dialogue between agent and voice.
Self Voice is additionally grounded in the user's current `self/soul` identity
layer through `python -m memory soul prompt self`, so the user's incorporated
principles can shape what the voice hears over time.

Soul Mode can now mature one provisional fruit. `python -m memory soul fruit set`
stores and renders `❦ FRUIT IN MATURATION`; a later `fruit set` replaces the
same fruit rather than accumulating disconnected takeaways. When the user asks to
harvest, `python -m memory soul harvest set` renders `❦ HARVESTED FRUIT` and asks
`save to journal?`. Only explicit confirmation calls `harvest save`, which
creates one journal entry and clears the harvested state. Decline clears the
state without saving.

The journal path became richer during validation. Soul Mode harvests are now
structured Markdown entries with a title, fruit block, origin metadata, and when
available a link back to the source conversation. The web app now exposes Journal
as its own pinned memory category, opens memory cards as object detail pages,
renders journal Markdown with headings, blockquotes, lists, code, and safe
internal links, and can navigate journal origin links back to the source
conversation.

Pre-release usage also produced several web and maintenance refinements: full
turn deletion in conversation detail, formatted journal cards, top navigation for
Workspace, Conversations, Journeys, Operations, Configuration, Preferences, and
Docs, and styling fixes for conversation journey assignment.

## Conscious Exclusions

- Wisdom Voice and Beauty Voice can appear as Possible Listenings, but they do not yet open full rites.
- Passagem remains a future rite, not part of this first release.
- Return To Center and Closing Day rituals remain future work.
- Soul Mode does not yet enrich Self, Shadow, Ego, persona, or journey identity layers after harvest.
- No automatic psyche mutation exists. Future identity enrichment must be proposed with visible diff and explicit confirmation.
- No rich web ritual UI is included. The first release validates textual ritual grammar.
- Soul Mode does not execute implementation work. Operational mutation requests must route to Builder Mode.

## What We Learned

The most important product correction was that voices should be heard, not
conversed with. When Self Voice or Shadow Voice behaved like a separate
interlocutor, the user could lose track of whether they were speaking to Mirror
or to the voice. The cleaner grammar is: the user always converses with Mirror;
Mirror listens through a voice; the voice appears in a card; Mirror bridges the
voice back to the conversation.

Self Voice also became clearer through use. It should not help solve the problem.
It should mirror what is not being seen, using the user's incorporated principles
to reveal what is happening. That distinction matters because Soul Mode is not a
coaching workflow. It is a ritual listening mode.

The harvest boundary also became sharper. The journal is not a transcript sink.
It receives one chosen fruit only after the user asks to harvest and confirms
saving. The web app then needs to treat that entry as a meaningful record, not as
plain text hidden inside a generic memory category.

## Next Horizon

The next Soul Mode horizon is integration. A future release should add a Closing
Rite that reviews harvested material and asks what wants to remain in the
Mirror's living psyche: Self principles, Shadow listening hypotheses, Ego
behavior, persona refinements, or journey memory. That work belongs after this
first ritual release because it touches identity mutation and must be designed
around explicit confirmation.
