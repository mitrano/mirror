# v0.17.3 — Builder Guard Non-Mirror Projects

**Date:** 2026-05-28

## Highlights

- Restricts the Builder clone-role guard to Mirror Mind source checkouts.
- Allows non-Mirror journey projects without `.mirror-clone-role` to load Builder Mode.
- Keeps protection for Mirror Mind project paths marked `production`.
- Updates boundary docs to clarify that the marker belongs to Mirror Mind clones, not all journey projects.

## Where We Started

v0.17.2 correctly moved Builder's clone-role inspection from the current runtime
checkout to the selected journey `project_path`. Production validation then
showed the remaining problem: ordinary projects without `.mirror-clone-role` were
still treated as `production` because the clone-role inspector defaults missing
markers to production.

That default is right for Mirror Mind source checkouts, where safety should be
conservative. It is wrong for arbitrary journey projects. Software Zen pointed to
`/Users/alissonvale/Code/szen_play`, a normal project checkout, and Builder still
refused because there was no Mirror clone-role marker.

## What Changed

Builder now first checks whether the target path appears to be a Mirror Mind
source checkout. Only then does it apply `inspect_clone_role()` and enforce the
production/dev boundary. Non-Mirror journey projects skip the Mirror-specific
clone-role guard and proceed normally.

If a journey points at Mirror Mind itself, the guard still works: production
Mirror checkouts refuse Builder by default, dev checkouts proceed, and
`--ignore-production-role` remains the explicit escape hatch.

## Conscious Exclusions

This patch does not introduce a generic production/dev marker for all possible
user projects. The existing `.mirror-clone-role` marker remains a Mirror Mind
runtime/source safety mechanism. Projects that want their own environment guards
should define them in their own tooling.

## Next Horizon

Production should update to v0.17.3 and retry Builder Mode for `softwarezen`.
The expected result is that `/Users/alissonvale/Code/szen_play` loads without
