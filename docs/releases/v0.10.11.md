# v0.10.11 — Pending Notes Prompt Validation Stub

**Date:** 2026-05-25

## Highlights

- Published a minimal stable release after the v0.10.10 prompt-routing patch.
- Intentionally contains no functional runtime changes.
- Exists to validate the final-user flow: welcome detects a newer stable release,
  suggests pending release notes, and the user's question renders cumulative
  release notes.

## Why This Patch Exists

v0.10.10 changed the welcome update prompt from the ambiguous `what changed?` to
`show pending release notes` and updated the release-notes skill to prefer the
cumulative pending view for newly available versions.

To validate that behavior in production, a newer stable release must exist while
production is already running v0.10.10.

## What Changed

Only the package version, release note, release index, and worklog move forward.
Runtime behavior is unchanged.

## Next Horizon

`show pending release notes`. Then ask that suggested question and confirm the
answer uses `runtime release-notes pending` instead of git status.
