# v0.17.2 — Builder Guard CI Formatting

**Date:** 2026-05-28

## Highlights

- Preserves the v0.17.1 Builder project-path guard behavior.
- Applies repository formatting expected by CI.
- Advances the package and release metadata so stable can point at a green patch commit.

## Why This Patch Exists

v0.17.1 published the intended Builder Mode fix, but the main CI run failed at
`ruff format --check` because a few files were not formatted according to the
repository-wide formatter gate. The runtime behavior was correct, but the release
should not leave the latest stable channel on a commit whose CI is red.

## What Changed

This patch formats the affected Python files and advances release metadata from
v0.17.1 to v0.17.2. The Builder guard semantics are unchanged from v0.17.1:
Builder inspects the selected journey's `project_path` clone role and falls back
to the current directory only when no project path is configured.

## Conscious Exclusions

This release does not add new Builder behavior, runtime update behavior, schema
changes, or web functionality.

## Next Horizon

Validate GitHub Actions for the v0.17.2 main push, then use the normal stable
update path from production.
