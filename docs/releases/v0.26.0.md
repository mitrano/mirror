# v0.26.0 — Soul Mode Integration

**Date:** 2026-06-09

## Highlights

- Adds the Soul Mode Closing Rite.
- Adds the multi-layer `☾ INTEGRATION PROPOSAL` surface.
- Makes post-closing integration explicit: living material may remain as part of Mirror identity only if the user chooses to integrate it.
- Keeps Integration Proposal proposal-only: `proposal only — nothing changed`.
- Supports confirmed Self, Shadow, Ego, and persona integrations through `memory soul apply`.
- Requires `--confirm APPLY` before any identity mutation.
- Preserves existing identity content by default.
- Adds atomic `identity_integrations` records for confirmed integrations.
- Appends confirmed integrations under layer-specific sections instead of replacing whole identity documents.
- Keeps journey identity out of Soul Mode Integration.

## Where We Started

Soul Mode could already open a ritual, hear multiple voices, mature a fruit, and
save a harvest to the journal. What was still missing was the threshold after
harvest: a way to close the field, see what remained alive, and decide whether
anything should become part of Mirror's identity without turning Soul Mode into an
automatic identity editor.

The first design separated review, proposal, and apply too sharply. In real
validation, the review surface already felt like a proposal. The release
therefore collapsed the review/proposal step into a single multi-layer
Integration Proposal and kept mutation as a separate, explicit confirmation step.

## What Changed

Soul Mode now has a Closing Rite. After a harvest is saved, Mirror can render a
`☾ CLOSING RITE` card that gathers what was harvested, what still echoes, what
remains open, and what may want integration. The rite is not itself a journal
save and not identity mutation. It is a threshold surface.

After closing, Mirror asks:

```text
There is living material that may want to remain as part of your Mirror identity. Do you want to integrate it now?
```

If the user says yes, Mirror renders `☾ INTEGRATION PROPOSAL`. The proposal may
include origin, Self, Shadow, Ego behavior, persona, and leave-open material. The
identity sections are directly registerable, while origin and leave-open remain
context. The card ends with:

```text
proposal only — nothing changed
```

The post-proposal prompt is direct:

```text
Do you want to record it this way in your identity? Or we can adjust anything you want.
```

If the user wants adjustment, Mirror should render an adjusted Integration
Proposal before applying anything. If the user confirms, Mirror applies only the
exact confirmed text for the selected identity sections.

Confirmed apply is now additive and provenance-preserving. The new
`identity_integrations` table stores each confirmed integration as an individual
record with layer, key, content, origin, optional conversation/journal provenance,
status, and metadata. The prompt-facing identity document remains the current
synthesized layer, but it is no longer overwritten by a short Soul Mode fragment.

Confirmed integrations append under layer-specific sections:

```text
self    → ## New Incorporated Principles
shadow  → ## New Hidden Needs Recognized
ego     → ## New Operational Patterns Identified
persona → ## New Participation Patterns Revealed
```

This release also tightened the ritual contract for Fruit In Maturation. New
material should thicken the fruit instead of replacing it with the latest insight.
The fruit remains singular in form but plural in root: one living phrase that can
carry multiple elements from the lineage of the session.

Finally, Self integration language was made more affirmative. Self proposals
state adopted first-person principles, not mere possibilities. For example:

```text
I care for bonds without turning immediate availability into moral proof of love; my inner measure also belongs to care.
```

## Conscious Exclusions

- No automatic identity mutation.
- No journey identity or journey-pattern integration.
- No replacement-mode identity mutation.
- No automatic re-synthesis engine for identity documents.
- No rich Web Soul Mode UI.
- No Passagem.
- No Return To Center.
- No Closing Day or Daily Soul Ritual.
- No new Soul voices.
- No local Wisdom/Beauty source library.

## What We Learned

The integration surface works best when it is already a proposal, not a neutral
review followed by another proposal. Once the user sees Self, Shadow, Ego, and
persona material together, asking for a second proposal introduces friction and
uncertainty. The right boundary is not review versus proposal; it is proposal
versus mutation.

We also learned that safe identity mutation requires two representations. The
identity `content` field is the current synthesized prompt-facing document, but
it should not be the only record of what entered the psyche. Individual
integration records preserve provenance, make future review possible, and prevent
short ritual insights from erasing long-standing identity material.

The fruit-maturation refinement clarified the ritual grammar: maturation is not
selection of the newest insight. It is condensation. A mature fruit can carry
more than one root while remaining one phrase.

## Next Horizon

The next Soul Mode horizon is the web experience: making the ritual and its
integration records visible in a richer interface without weakening the
confirmation boundary. Future work may also add identity re-synthesis from
integration records, but replacement and automatic consolidation should remain
explicit, inspectable operations rather than hidden Soul Mode behavior.
